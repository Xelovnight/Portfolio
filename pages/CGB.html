Présentation de CGB.

CGB est une application banquaire, qui permet de gérer les virements entre plusieurs comptes etc... C'est une application JAVA dans un environnement Spring / MAVEN.

Décortication et Explication.

Voyons comment est composé CGB.

Base de donnée H2

Dans le projet CGB, nous avons utilisé une base de donnée H2 ce qui nous à permis de développer en NOSQL. C'est donc cette tâche qui m'a permis de comprendre et de mettre en place pour le premiere fois du NOSQL.

API REST

Pour le projet CGB, j'ai suivi le modèle Repository, Service, et Controller comme convennu dans un projet SPRING. En partant de cette base, j'ai pu créer une API REST qui me permet de gérer les utilisateurs, les comptes, et les virements entre ces derniers. Pour tester cette API REST, j'utiliser POSTMAN pour simuler les requêtes et les réponses de l'API.

Les tests

Pour le projet CGB, j'ai utilisé JUNIT pour tester les différentes classes et méthodes de mon projet. J'ai donc pu créer des tests unitaires pour chaque méthode de mes classes, et ainsi m'assurer que tout fonctionne correctement. J'ai également utilisé Mockito pour simuler les appels à la base de données et tester les méthodes de mes classes sans avoir à dépendre de la base de données.
